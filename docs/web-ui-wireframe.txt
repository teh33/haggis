Haggis Web UI Wireframe
=======================

Visual direction
----------------

Match kf.cafe:
- black background
- white terminal chrome
- monospace text
- sharp square panels/buttons/cards
- minimal text in the play area
- debugging/log details collapsed by default

Card direction:
- Do not use suit symbols as the primary card identity.
- Use four card colors instead:
  - Red
  - Cyan
  - Green
  - Yellow
- Card numbers/ranks should be much larger than they are now.
- Show point value as repeated `*` marks instead of numeric point labels:
  - J = `**`
  - Q = `***`
  - K = `*****`
  - ordinary non-point cards = no stars
- Wild cards should still be visually marked, but not by tiny point text.

Example card rendering
----------------------

Possible actual color treatment:
- Keep cards black with white outlines.
- Add a strong color strip, corner tag, or glow per suit/color.
- Rank remains white and large for readability.
- Stars can be white, or inherit the card color if that reads better.

Drag/drop behavior:
- Cards can be reordered before betting.
- While dragging, show a bright vertical insertion line before/after the target.
- Sort by number orders by rank, then color.
- Sort by suit/color groups by Red, Cyan, Green, Yellow, then rank.

Implementation notes from iteration
-----------------------------------

Already implemented:
- browser UI served by `python3 -m haggis.web`
- deployed Haggis service on Render
- `/haggis` page on kf.cafe embeds the Render service
- kf.cafe terminal theme applied to the game UI
- game to 350
- CPU wilds visible
- current play cards displayed as cards
- stack clears when CPU/player passes
- end-of-hand scoring breakdown
- drag/drop hand ordering
- sort by number and sort by suit

Next visual change requested:
- Replace suit-symbol identity with Red/Cyan/Green/Yellow identity.
- Increase card rank size substantially.
- Replace numeric point labels with repeated stars.
